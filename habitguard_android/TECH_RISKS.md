# HabitGuard Technical Risks

Risk / Severity / Evidence / Impact / Mitigation / Current status

Android UsageStatsManager special permission and event retention / High / `AndroidManifest.xml` declares `PACKAGE_USAGE_STATS`; `UsageStatsRepository.hasUsageAccess()` checks AppOps; `UsageEventAggregatorTest` covers JVM aggregation edge cases / Users may not grant access and OEM event retention can vary, especially across reboot or long idle periods / Keep permission explanation, store `UsageDataQuality`, distinguish permission loss from zero usage, run connected-tablet tests / Partially implemented; event aggregation tests pass, real-device results not recorded.

Accessibility Service policy and reliability / High / `HabitGuardAccessibilityService.kt`; `RestrictionEvaluator.kt`; `habitguard_accessibility_service.xml` uses package/window events and `canRetrieveWindowContent=false` / Google Play review risk; app interruption can miss events; not true OS-level blocking / Describe as user-approved interruption flow, keep least-privilege config, test Guard v2 on the connected tablet / Partially implemented; JVM evaluator tests pass, device flow not recorded.

Accessibility content minimization / High / Static check found no `rootInActiveWindow`, `AccessibilityNodeInfo`, `event.text`, or `event.contentDescription`; XML sets `canRetrieveWindowContent=false` / Reading screen content or input text would create major privacy risk / Keep current no-content-read policy and enforce with `scripts/security_check.py` / Implemented for current code.

Notification access / High / `HabitGuardNotificationListenerService.kt` stores package/date/count only / Notification permission is sensitive; body/title collection would be excessive / Do not read notification extras/title/body; keep count-only storage; verify on device / Partially implemented.

HTTPS-only networking / High / `AndroidManifest.xml` sets `usesCleartextTraffic=false`; `network_security_config.xml` base config has `cleartextTrafficPermitted=false` / Cleartext HTTP could expose usage summaries or tokens / Keep HTTPS-only config and static check / Implemented.

Cloud sync and Firestore auth / High / `FirestoreSyncRepository.kt` uploads only after consent; bearer token is read through `SecureTokenStore`; no auth/config UI exists / Sync cannot work for normal users yet; tokens remain sensitive / Add proper auth/config UI or hide sync controls; never commit tokens / Partially implemented.

Local Room data loss / High / Device `SM-F766N` had `habitguard.db` at `user_version=7`; launch crashed with `A migration from 7 to 11 was required but not found`; `HabitGuardDatabaseMigrations.MIGRATION_7_11` and `MIGRATION_10_11` are now registered and migration tests passed on device when filtered / Future schema changes can still risk local goals, rules, usage snapshots, mission logs, unlock sessions, and predictions if a migration is omitted or incorrect / Keep destructive fallback disabled, require explicit migrations from observed installed versions, preserve representative user rows in migration tests, and follow `ROOM_MIGRATION_POLICY.md` for any non-preserving change / Mitigated for observed 7-to-11 and 10-to-11 paths; full connected test suite still has an unrelated Compose test failure.

Local privacy and backup / Medium / `allowBackup=false`; backup/data extraction rules exclude `database` and `sharedpref`; local Room stores usage metadata and mission/goal text / Local private metadata remains on device and in user-exported CSV / Keep local-first defaults, add clearer deletion controls, avoid committing private exports / Partially implemented.

Free-form text in cloud payload / Medium / `rawText` removed from Firestore summary payload; local Room still stores goal raw text / Free-form text could include private user notes / Keep raw text local unless explicitly needed and consented / Mitigated for current cloud payload.

Debug build attack surface / Medium / `app/src/debug/AndroidManifest.xml` exports `DebugExportReceiver` for ADB CSV export / Debug builds expose test-only receiver and should not be distributed / Keep receiver debug-only; do not ship debug APKs as production / Partially mitigated.

Firebase SDK / Medium / No Firebase Gradle dependency or `google-services.json`; only Firestore REST code exists / Firebase integration claims would be overstated / Say Firestore REST skeleton only / Not implemented.

FastAPI / Cloud Run backend / Medium / No FastAPI app, server routes, Dockerfile, or Cloud Run config under this project / Backend/deployment claims would be unsupported / Keep as future work unless implemented and tested / Not implemented.

Model reliability / High / Current `training_manifest.json`, `habitguard_model_profile.json`, and `android_inference_bundle.json` have `source_type: synthetic` and `evaluation_scope: synthetic evaluation`; Android executes exported Linear/Logistic model math locally through `TrainedLocalPredictionModel`, and WorkManager now stores offline prediction results / Synthetic metrics and coefficients may not generalize to real users even though Kotlin math matches the Python fixture / Label synthetic results as pipeline verification only; train on approved real export before real-user claims; keep 7-day collecting-data gate and show synthetic caveat / Local-first inference is implemented; real-user validity is unresolved.

Prediction result integration / Medium / `PredictionRepository.kt` separates remote, local, baseline, cache, and collecting-data states; `ModelBundleLoader.kt` validates the local JSON bundle; `DailyUsageSyncWorker.kt` refreshes offline predictions through `LocalFirstPredictionRefreshEngine`; `HabitGuardApp.kt` displays source, model version, calculation time, data quality, `source_type`, `evaluation_scope`, risk, and recommendation / Users may over-trust synthetic local-model estimates if copy is unclear / Keep approximate 10-minute rounding, show synthetic caveat, keep collecting-data state below 7 complete days, and avoid auto-applying rules from prediction output / Implemented for local-first offline prediction; real-user validity and device UI screenshot remain unresolved.

Prediction backend privacy / High / `PredictionRequestDto.from()` uses daily summaries and category totals only; no production remote endpoint is configured / A future backend could accidentally receive raw app timeline, notification body, package names, or free-form text if DTO boundaries are bypassed / Keep DTO tests, require auth/user authorization/rate limits/environment separation before backend deployment / Interface implemented; backend not implemented.

Data bias and synthetic sample risk / High / `ai/train_from_phone_csv.py` creates deterministic synthetic sample if no CSV is supplied; generated artifacts include `data_snapshot_hash.txt`; root synthetic dataset marks `source_type=synthetic` / Generator assumptions can dominate results even when models beat baselines / Keep `source_type` explicit, reject mixed source types, avoid random row 80:20 splits, and do not mix synthetic and real claims / Partially mitigated for pipeline integrity; unresolved for real-world validity.

User approval for restrictions / High / Rules are persisted only through `GoalScreen` save path and `HabitGuardViewModel.saveRule`; `RestrictionRuleDao.enabledRuleFor()` requires `approved = 1 AND enabled = 1`; evaluator tests cover unapproved/disabled rules / Auto-applying restrictions would be coercive and unexpected / Keep explicit confirmation and avoid background rule activation / Implemented for current flow, with JVM evaluator coverage.

Android tests and regression risk / Medium / `UsageEventAggregatorTest.kt`, `RestrictionEvaluatorTest.kt`, `PredictionRepositoryTest.kt`, `DashboardPredictionPresenterTest.kt`, `ScreenTimingComposeSemanticsTest.kt`, and `RoomMigrationVerificationTest.kt` exist; `scripts/quality_gate.py` runs the automated local gate / Parser, mission UI validation, explicit Room version migrations, Firestore payload, and real AccessibilityService/LockActivity flows can still regress / Add tests for remaining core behaviors and run connected instrumentation tests on Samsung/Pixel devices / Partially implemented.

Screen Timing UI accessibility display modes / Medium / `docs/screen_timing_design/captures/*-uiautomator.xml` confirms labels/content descriptions on a connected tablet, but manual TalkBack, 200% font size, landscape, and dark-mode captures were not completed / Long Korean text or high font scale could still clip or require extra scrolling / Add Compose semantics instrumentation tests and manual QA captures for 200% font, landscape, dark mode, and TalkBack reading order / Partially implemented.

CI and static analysis / Medium / `.github/workflows/android-quality.yml` runs `python scripts/quality_gate.py`; local quality gate covers formatting, security check, Python ML tests, JVM tests, androidTest APK build, lint, and debug build / GitHub-hosted CI does not run connected-device instrumentation and no Detekt/Ktlint config exists / Keep CI gate required before sharing and add device-lab/manual QA evidence separately / Partially implemented.

Room migration coverage / High / `RoomMigrationVerificationTest.kt` creates version 7 and version 10 databases, inserts representative user rows, opens them through the runtime Room builder, and validates preserved rows/defaults; filtered `connectedDebugAndroidTest` passed on `SM-F766N` / The full connected test suite still fails in an existing Compose Semantics test, so migration coverage is not the only device quality gate / Extend the migration test every time entities change and fix the Compose instrumentation launch issue separately / Implemented for migration paths currently observed; broader device QA remains incomplete.

iOS Screen Time APIs / Medium / No iOS implementation under this Android project / iOS claims would be unsupported / Treat iOS as separate future work / Not implemented.

Google Play review / High / UsageStats, AccessibilityService, notification listener, and lock/mission behavior are sensitive / Store review may reject or require stronger disclosure / Prepare policy language, privacy policy, and real-device evidence; avoid medical/full-blocking claims / Unknown.

App Store review / Medium / No iOS implementation in this project / App Store claims are out of scope / Do not claim iOS availability / Not implemented.
